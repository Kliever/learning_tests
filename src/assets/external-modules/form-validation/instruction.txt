ПОСЛЕДОВАТЕЛЬНОСТЬ ПРОВЕРОК: Сначала проверяет все инпуты на наличие символов, затем чекбоксы на чеккед, затем поле пароля и поле подтверждения пароля. Затем поле ввода пароляпроверяется на наличие запрещенных символов, а следом на минимально допустимое количество символов. Если проходит, проверяется на совпадение оба поля пароля. 


Поле пароля и поле подтверждения пароля может быть только одно

.js-form-error - класс необязательного попапа, который высвечивается при возникновении ошибки заполнения формы. Если попапа с этим классом нет, плагин будет полноценно работать, но сообщения об ошибке в виде поп-апа высвечиваться не будут.

.js-form-valid-submit - класс поля подтверждения пароля. Если такое поле существует, будет проводиться проверка на соответствие значению поля пароля при отсутствии прочих ошибок. Класс использовать необязательно.


Атрибуты для .js-form-valid:

data-timeout - необязательный атрибут, устанавливается тегу формы (класс .js-form-valid), время в секундах, через которое класс _error снимается с объектов самостоятельно. При отсутствии атрибута _error будет сниматься сразу при заполнении поля или установки чек на чекбокс


Атрибуты для .js-form-valid-pass:

data-limit="4" - Необязательный атрибут, указывающий минимально допустимую длинну пароля (включительно). Если атрибут или его значение не указывать, можно вводить пароль бесконечной длины

data-latin - Необязательный атрибут для класса пароля (.js-form-valid-pass), позволяет использовать в качестве пароля только латинские буквы и цифры. Если не указывать, будут использоваться любые символы клавиатуры, если указать сам атрибут, но оставить его пустым, для ввода будут разрешены только цифры и латинские буквы. В значении можно указать без пробелов (подряд) символы, которые разрешены дополнительно.
